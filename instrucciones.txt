Simular Backend

Todos los servicios creados pueden ser dummy (Simulados) para efectos de simular una petición http, se pueden utilizar cualquier herramienta online disponible como “Mocky: The world easiest & fastest tool to mock your APIs “

Crear un API de usuarios que retorne un token JWT, este servicio debe recibir por post user y password

Generar un servicio con una operación CRUD (Create, Read, Update y Delete), este servicio puede ser de la entidad de tu preferencia, ejemplo (Vehículos, Bicicletas, Notebook, Consolas de videojuegos, etc. etc. etc.).

Frontend

Generar un app en React (no jQuery).

Crear una vista de login, donde consuma el servicio dummy que retorna el token, la cual recibe user y password.

Luego de autenticarse, se debe redirigir a una vista, donde se consuman los servicios CRUD de la entidad seleccionada por el postulante, esta vista debe contener una tabla con las diferentes acciones, como por ejemplo:


Cada petición realizada a los servicios debe contener en la cabecera el parámetros “Authorization“ con el valor del token retornado por el servicio de usuario.

Todas la peticiones deben enviar y recibir JSON.

Lineamientos Generales

El proyecto debe ser subido a un tu repositorio git, el cual debes compartir.

El proyecto debe contener un archivo README.md, donde describa paso a paso, cómo levantar el proyecto, es permitido también, ambientes contenerizados (Docker).

Te recomendamos si no entiendes algún punto que consultes y con gusto te apoyamos.
